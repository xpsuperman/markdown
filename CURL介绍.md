# CURL介绍

`curl命令`是一个利用URL规则在命令行下工作的文件传输工具。它支持文件的上传和下载，所以是综合传输工具，但按传统，习惯称curl为下载工具。作为一款强力工具，curl支持包括HTTP、HTTPS、ftp等众多协议，还支持POST、cookies、认证、从指定偏移处下载部分文件、用户代理字符串、限速、文件大小、进度条等特征。做网页处理流程和数据检索自动化。

####查看页面源码

```shell
##直接在curl命令后加网址，就可以看到网页源码

```

```shell
##使用-L参数，curl就会跳转到新的网址

```

#### 发送表单信息

```shell
##发送表单信息有GET和POST两种放，GET方法想到简单，只要吧数据负载网址后面就行。
curl example.com/form.cgi?data=xxx
#POST方法必须把数据和网址分开，curl就要用到--data或者-d参数
curl -X POST --data "data=xxx" example.com/form.cgi
#如果你的数据没有经过变淡编码，可以让curl为你编码，参数是--data-urlencode
curl -X POST --data-urlencode "date=April 1" example.com/form.cgi
```

#### HTTP动词

```shell
#curl 默认的http动词是GET,使用-X参数可以支持其他动词
curl -X POST www.example.com
curl -X DELETE www.example.com
```

#### User Agent字段

```shell
#该字段标识客户端的设备信息，服务器会根据这个字段，针对不同设备，返回不同格式的网页
curl --user-agent "[User Agent]" [URL]
```

#### cookie

```shell
#使用 --cookie 参数，可以让curl发送cookie
curl --cookie "name=xxx" www.example.com
#至于具体的cookie的值，可以从http response头信息的Set-Cookie字段中得到。
```

#### 增加头信息

```shell
#有时需要在http request之中，--header增加一个头信息
curl --header "Content-Type:application/json" http://example.com
```

#### HTTP认证

```shell
#有些网域需要HTTP认证，这是curl需要用到--user或者-u参数
curl --user name:password example.com
```
